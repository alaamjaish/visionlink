Three processes on the Pi

Independent processes give isolation: a crash in one cannot kill the others. audio_worker is a subprocess of FastAPI, so a C-level ALSA assertion crashes only the worker — the parent restarts it within 1.5 s.

Process 1

GPIO Bridge

scripts/gpio_bridge.py

Owns the six GPIO pins. Classifies each gesture as single, double, hold_start, or hold_end, and POSTs to /api/button/<n>/<event>.

Process 2

FastAPI Dashboard

dashboard/server.py · port 8000

Voice command center, WebSocket transcript, camera lock manager, hosts the audio_worker subprocess. ~1 800 lines.

Process 3

Ops Dashboard

ops/ · port 3000

Next.js 16 + React 19. Speaks directly to Supabase via @supabase/supabase-js for realtime panels and CRUD.

Three AI providers

Each picked for a specific strength. All three can be live in the same wearable; the supervisor switches between them from the Wearable Settings panel.

Default · B4 / B5 / SOS

Google Gemini Live

Models: gemini-3.1-flash-live-preview (voice + vision streaming) and gemini-2.5-flash (text fallback).

Why: multimodal native (audio + image + text in one stream), fastest first-token latency among the three, lowest cost per minute.

Override · B4 / B5

OpenAI Realtime

Model: gpt-realtime-2. SOS path uses voice "marin" with reasoning effort medium.

Why: stricter instruction-following for tightly-scripted personas. Best perceived "presence" on high-stakes flows. Output speed 1.2× by default to combat the model's natural pacing.

Chatbot · this site

Gemini 3.5 Flash

Model: gemini-3.5-flash (released 2026-05-19).

Why: the newest Flash-tier model — 4× faster than 3.1 Pro on agent and coding benchmarks, $1.50 / $9 per 1 M tokens. Powers the chat bubble at the bottom-right of this page.

The six AI tools

Both Gemini and OpenAI sessions expose the same six tools via a shared TOOL_DECLS and TOOL_HANDLERS registry. Without tools the AI could only talk; with them, it can do.

#ToolWhat it doesTables
1lookup_component(query)Search components by part code (exact) then name (fuzzy). Returns up to 3 rows with torque spec, maintenance interval, safety notes.Read
2log_incident(description, category?, severity?, location?)Insert into incidents. Categories: safety / equipment / leak / damage / other. Severities: low / medium / high / critical.Write
3mark_task_complete(task_query)Fuzzy-match a pending task by title fragment → mark complete. Returns ambiguous=true with options when multiple match.Write
4get_my_assignments(include_complete?)List the worker's tasks, ordered by priority then due date.Read
5request_part(part_query, quantity?, urgency?, reason?)Submit to part_requests. Urgencies: normal / urgent / critical. Auto-matches against components for known codes.Write
6send_report(report_name, recipient_role?, recipient_name?, recipient_email?, custom_message?)Look up manager + template → inject live data → send via Gmail SMTP (default) or Resend → audit-log to sent_reports.R + W + email
Three absolute rules in the system prompt prevent the most common failure modes:
1. Never lie about completing actions — confirmation requires the dispatcher's function_call_output echo carrying the actual row ID.
2. Verb triggers — log, mark, send, request in the worker's speech fire the matching tool immediately.
3. The AI may ask at most one clarifying question, otherwise commits with reasonable defaults.

The Ops Dashboard

Next.js 16 + React 19 + Tailwind 4. Two pages, eleven realtime panels. New rows appear in the UI within ~200 ms of being written.

/ — Command Center

  • SosPanel — live SOS alarm + supervisor shutdown
  • WearableSettingsPanel — remote config (provider, vision mode, SOS tunables)
  • IncidentsPanel — safety + equipment logs
  • TasksPanel — worker task assignments
  • PartsPanel — part requests
  • ComponentsPanel — factory parts catalog
  • ManagersPanel — supervisor directory
  • ReportTemplatesPanel — email templates
  • SentReportsPanel — email audit trail

/documentation

  • Stats strip — total / open sessions, photos, videos, voice notes, orphans
  • SessionsPanel — list of shifts with start/end, duration, per-kind asset counts. Click a row to expand and see every capture in that session.
  • CapturesPanel — tabbed grid (All · Photos · Videos · Voice notes · Orphans). Photos open in a modal preview. Videos use native <video> controls. Voice notes use native <audio>.
