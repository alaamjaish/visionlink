### D. Major bugs found and fixed
1. **`session.receive()` was exiting after one turn** ‚Äî root cause of "Gemini only replies once then dies". Fixed by wrapping in an outer `while True:` loop. `session.receive()` is per-turn, not per-session.
2. **Echo feedback loop** ‚Äî Gemini's speaker output leaking into the mic, causing spurious `interrupted` events on its own turn. Fixed with half-duplex: mic send is muted while the speaker queue has pending Gemini audio.
3. **Speaker blocking the receive loop** ‚Äî blocking `spk_stream.write()` inline stalled the `async for` loop. Fixed with a dedicated player task reading from an `asyncio.Queue` and writing via `asyncio.to_thread`.
4. **`interrupted` events weren't flushing the queue** ‚Äî user couldn't barge in. Fixed: on `interrupted`, we drain the queue immediately.
5. **Tuned VAD** ‚Äî set `start_of_speech_sensitivity=LOW`, `end_of_speech_sensitivity=LOW`, `silence_duration_ms=800` in `types.AutomaticActivityDetection`. Less twitchy on the I2S mic's noise floor.
6. **Port 8000 binding issues** ‚Äî `pkill` in the same Bash call was killing the parent shell; switched to a cleaner kill-then-launch pattern.

## 4. Key technical decisions (and why)

| Decision | Chosen | Rejected | Why |
|---|---|---|---|
| Voice model | `gemini-3.1-flash-live-preview` | OpenAI `gpt-realtime`, Anthropic voice | Free in preview, WebSocket duplex works, we already have it running |
| Writer subagent | Claude Sonnet 4.6 (called as a tool from Gemini Live) | Gemini itself writing freely, OpenAI GPT-5, Google ADK delegation | Claude scored 99.8% on structured-output compliance in 2026 benchmarks ‚Äî highest of any frontier model |
| Button 5 retrieval | Direct `supabase.table(...).ilike(...)` | pgvector embeddings, hybrid BM25+vector | Only 7‚Äì8 components; latency-sensitive voice UX; embeddings are overkill |
| Email delivery | Resend (free 100/day) | Gmail SMTP, AWS SES | Resend has the cleanest Python SDK and no card required |
| Echo handling | Half-duplex mute during Gemini playback | PulseAudio module-echo-cancel, WebRTC AEC | Half-duplex is bulletproof and what Google's own Pi reference does; AEC can come later |
| Audio lib | `sounddevice` | PyAudio | PyAudio has known failures on Bookworm; `sounddevice` has clean ARM64 wheels |
| Git auth on Pi | `gh auth login --web` device code | PAT paste, SSH keys | One code, one browser click, done |
